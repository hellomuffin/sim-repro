BEHAVIOR-1K — capability reference

A human-centered embodied-AI benchmark of 1,000 everyday household activities simulated with full rigid-body & fluid physics. Unlike symbolic household simulators, BEHAVIOR's agent is a real robot that manipulates objects whose physical and chemical state changes — food gets cooked, sliced, soaked, frozen, burnt; cloth gets folded; surfaces get wiped. We document its interaction set and render the robot's egocentric camera.

Motivation

Why BEHAVIOR — a much larger interaction space

Symbolic simulators like VirtualHome expose a small set of atomic actions (grab, put, open, switch). BEHAVIOR's agent performs physically-simulated interactions whose effects are governed by real state machines and transition rules, so the action vocabulary is both larger and grounded — a "cut" actually produces slices, a "soak" actually saturates a sponge.

Interaction set · the headline

Action primitives

OmniGibson ships two semantic action-primitive sets. The Symbolic set (14 primitives) is the full interaction vocabulary; the Starter set (9) is the motion-planned subset used for IK baselines. Each primitive is a time-extended skill that expands into low-level robot control via a sampling-based / cuRobo motion planner. What makes the actions real

Transition rules & recipes

Complex physical/chemical changes that Omniverse doesn't natively support are implemented as transition rules — state machines that fire when preconditions hold (heat, contact, tool, container) and rewrite the scene (spawn slices, convert fluids, produce a cooked dish). This is the machinery behind CUT, SOAK, and heating. Source: omnigibson/transition_rules.py + bddl/generated_data/transition_rule_recipes.json

Rule familyEffect
SlicingRule / DicingRuleKnife + sliceable → spawn slices / diced particles
WasherRule / DryerRuleRemove/clean or dry covered particles & saturation
MeltingRuleHeat a meltable object past threshold → molten fluid
CookingRule · CookingObjectRule · CookingSystemRule · CookingPhysicalParticleRuleCook discrete objects, fluid systems, and particles
ToggleableMachineRuleA switched-on machine transforms its contents (blender, oven)
MixingToolRule · BlenderRule (RecipeRule)Recipe-based fluid/ingredient transforms (coffee, smoothie, soup…)

Recipes are data-driven: e.g. coffee_bean + water →(blender) drip_coffee, cappuccino, lemonade; cooking recipes for sugar_cookies, pizza, beef_stew, mac_and_cheese. These are exactly the multi-step goals BEHAVIOR tasks ask the robot to achieve.

1,018 BDDL activities

Each task is written in the BEHAVIOR Domain Definition Language (first-order logic): an :objects list, an :init scene state, and a :goal of predicates to satisfy.

e.g. brewing_coffee → goal (and (real ?drip_coffee) (contains ?mug ?drip_coffee)). Tasks span cooking, cleaning, tidying, assembling — the everyday activities the benchmark scores.
